  <action>
Add a `build` section to the existing vite.config.ts `defineConfig` call. Insert it after the `server` section:

```typescript
build: {
  rollupOptions: {
    output: {
      manualChunks: (id) => {
        // React core — changes rarely, largest cache benefit
        if (id.includes('node_modules/react') || id.includes('node_modules/react-dom')) {
          return 'vendor-react';
        }
        // Large icon library
        if (id.includes('node_modules/lucide-react')) {
          return 'vendor-icons';
        }
        // Date utilities
        if (id.includes('node_modules/date-fns')) {
          return 'vendor-date';
        }
        // QR libraries
        if (
          id.includes('node_modules/qrcode') ||
          id.includes('node_modules/html5-qrcode')
        ) {
          return 'vendor-qr';
        }
        // All remaining node_modules (including xlsx-js-style — Phase 2 removes it)
        if (id.includes('node_modules')) {
          return 'vendor';
        }
      },
    },
  },
},
```

Do NOT remove or modify the existing `plugins`, `server`, or any PWA config. Only ADD the `build` key alongside existing keys.

Per D-03: Do NOT remove xlsx-js-style from package.json — that is STAB-08 work. It will be included in the 'vendor' chunk which is acceptable for Phase 1.
