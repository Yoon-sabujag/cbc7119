## Key Behavior

- **Admin 가드:** `ctx.data.role !== 'admin'` → 403 `관리자 전용`.
- **VAPID 가드:** `ctx.env.VAPID_PUBLIC_KEY` 또는 `VAPID_PRIVATE_KEY` 누락 → 500 `VAPID 키가 서버에 설정되지 않았습니다`.
- **구독 조회:** `SELECT ... FROM push_subscriptions WHERE staff_id = ctx.data.staffId` → 0건이면 400 `등록된 푸시 구독이 없습니다. 먼저 구독을 켜주세요.`.
- **페이로드:** `{ title: '테스트 푸시', body: '수신 확인용', type: 'test' }`.
- **만료 처리:** 각 endpoint fetch 응답이 `410` 또는 `404`면 해당 구독 행을 `DELETE FROM push_subscriptions WHERE id = ?`로 자동 정리.
- **집계:** `{ success: successCount > 0, data: { sent, total, results } }` 반환, 전원 실패 시 `error: '모든 구독에 발송 실패'` 포함.
- **UI 가드:** `staff?.role === 'admin' && subscribed && permState === 'granted'` 세 조건 모두 참일 때만 버튼 렌더(비노출 가드). 비관리자는 UI에서 아예 보이지 않으며 API 호출도 403으로 거부된다.
- **전송 중 UX:** `testSending` state로 버튼 비활성화 + 라벨을 `⏳ 전송 중...`으로 교체해 중복 클릭 방지.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Build blocker] buildPushPayload body 타입 에러 해결**

- **Found during:** Task 1 (최초 `npx tsc --noEmit` 실행 시)
- **Issue:** `@block65/webcrypto-web-push`의 `buildPushPayload`가 반환하는 `body`가 Pages 타입 컨텍스트에서 `Uint8Array<ArrayBufferLike>`로 추론되어 `fetch`의 `BodyInit`와 overload mismatch 발생.
  ```
  error TS2769: No overload matches this call.
  Type 'Uint8Array<ArrayBufferLike>' is not assignable to type 'BodyInit'.
  ```
- **Fix:** `fetch` 호출 지점에서 `body: pushData.body as unknown as BodyInit` 로 캐스트. 런타임에서는 `Uint8Array`가 정상적인 `BodyInit`(ArrayBufferView)로 동작 — 타입 시스템의 과도한 좁힘만 해결.
- **Files modified:** cha-bio-safety/functions/api/push/test.ts
- **Commit:** 7dc69eb (Task 1 내)
