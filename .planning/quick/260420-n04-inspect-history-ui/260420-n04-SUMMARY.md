## Implementation Notes

### Task 1 — 서버 min_age TTL
- `url.searchParams.get('min_age')` 파싱 → 기본 21600, 음수 clamp 0, NaN→기본값
- `refresh=true & min_age>0`: `SELECT fetched_at … ORDER BY fetched_at DESC LIMIT 1` 로 최신성 체크. `Date.now() - fetched_at < min_age*1000` 이면 `syncOne()` SKIP → `loadFromDb()` → `cached:true, lastFetchedAt: latestFetch.fetched_at`
- `refresh=0` 경로: `SELECT MAX(fetched_at)` 조회 추가 → `lastFetchedAt` 포함
- `sync_all=1` 경로: `min_age` 완전 무시 (초기 관리자 동기화용 — 주석 한 줄로 명시)
- 3개 응답 경로 모두 `lastFetchedAt: string | null` 필드 포함 (grep count 3)

### Task 2 — 프론트 유틸
- 기존 `api.get<T>` 재사용 → Auth Bearer 주입 + 401 auto-logout 자동 처리
- `min_age` 파라미터는 명시하지 않음 (서버 기본값 사용)
- 4개 타입 export: `InspectFailItem`, `InspectHistoryItem`, `InspectHistoryResponse`, `fetchInspectHistory`
