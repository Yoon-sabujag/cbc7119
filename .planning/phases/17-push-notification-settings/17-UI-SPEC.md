## Interaction Contract

### Toggle ON interaction (D-10)

When user taps a notification toggle to turn it ON:

1. If `Notification.permission === 'default'`: call `Notification.requestPermission()` first
   - If user grants: subscribe via `PushManager.subscribe()`, then save preference ON
   - If user denies: do not flip toggle, show toast "알림 권한이 차단되었습니다. 브라우저 설정에서 허용해주세요."
2. If `Notification.permission === 'granted'`: directly save preference ON via API
3. If `Notification.permission === 'denied'`: do not flip toggle, show inline sub-label "브라우저 설정에서 알림을 허용해주세요" (var(--warn)), toggle rendered disabled (var(--bg4) background, not-allowed cursor)

### Toggle OFF interaction

When user taps a notification toggle to turn it OFF:
- Immediately update `notification_preferences` via PATCH API (no permission check needed)
- Toggle flips to OFF state optimistically, reverts on API error with toast "설정 저장에 실패했습니다. 다시 시도해주세요."

### Blocked permission state

When `Notification.permission === 'denied'`:
- All 6 notification toggles are rendered with `disabled` styling (var(--bg4) background)
- Tapping any toggle shows toast: "브라우저 설정에서 알림을 허용해 주세요."
- Permission status badge shows "차단됨" in var(--danger)

### Loading states

- Individual toggle rows show no loading indicator (optimistic update)
- On initial load, if preferences are being fetched, toggles render as OFF (disabled) until data resolves — no skeleton, matches existing SettingsPanel pattern of no loading states

## States Required

The following UI states must be implemented:

| State | Trigger | Visual |
|-------|---------|--------|
| Permission: default | `Notification.permission === 'default'` | Badge: 권한 미설정 (t3), all toggles enabled |
| Permission: granted, not subscribed | Permission granted but no push subscription in D1 | Badge: 허용됨 (safe), toggles enabled, all OFF |
| Permission: granted, subscribed | Active subscription exists | Badge: 허용됨 (safe), toggles reflect preferences |
| Permission: denied | `Notification.permission === 'denied'` | Badge: 차단됨 (danger), all toggles disabled (bg4, not-allowed) |
| Toggle ON pending | Awaiting requestPermission() resolution | Toggle in pressed state — no spinner |
| Preference save error | API PATCH failed | Toast error + toggle reverts |
