    ```typescript
    // After fetching schedRows (around line 34-40 area), compute completion for each schedule
    const todayScheduleWithCompletion = await Promise.all(
      (schedRows.results ?? []).map(async (r: any) => {
        let completed = false

        if (r.category === 'inspect' && r.inspection_category) {
          // Per D-17: date+category auto-matching via check_records JOIN check_points
          // Per D-18: 1+ check records = completed
          const cpCat = CATEGORY_ALIAS[r.inspection_category] ?? r.inspection_category
          const rec = await env.DB.prepare(`
            SELECT 1 FROM check_records cr
            JOIN check_points cp ON cr.checkpoint_id = cp.id
            WHERE cp.category = ?
              AND date(cr.checked_at) = ?
              AND cr.result IN ('normal','caution')
            LIMIT 1
          `).bind(cpCat, r.date).first()
          completed = !!rec
        } else {
          // Per D-19: non-inspect items use manual status='done'
          completed = r.status === 'done'
        }

        return {
          id:        r.id,
          title:     r.title,
          date:      r.date,
          time:      r.time ?? undefined,
          category:  r.category,
          status:    r.status,
          completed,
          memo:      r.memo ?? undefined,
        }
      })
    )
    ```

    Then replace the existing `todaySchedule` in the response (line ~164):
    ```typescript

    ```

    **Note on D-20 (multi-day inspection date range):** The basic implementation uses `date(cr.checked_at) = ?` (same-day match). For multi-day inspections (DIV 2-day, extinguisher 3-day), the full date-range logic (일정일~다음 같은 카테고리 일정일) is a refinement. Start with same-day matching — this handles the majority case. If time permits, extend the WHERE clause to use a date range by querying the next schedule_items row with the same inspection_category:
    ```sql
    -- Find next same-category schedule date
    SELECT date FROM schedule_items
    WHERE date > ? AND category='inspect' AND inspection_category = ?
    ORDER BY date ASC LIMIT 1
    ```
    Then use `BETWEEN r.date AND nextDate` instead of `= r.date`.

    **Important:** The `CATEGORY_ALIAS` map already exists in stats.ts. Reuse it — do NOT create a duplicate. Verify its contents match the categories used in schedule_items.inspection_category.
