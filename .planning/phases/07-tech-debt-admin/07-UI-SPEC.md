| Token | Value | Usage |

## Color

All values reference CSS variables defined in `src/index.css`.

| Role | CSS Variable | Hex Value | Usage |
|------|-------------|-----------|-------|
| Dominant (60%) | `var(--bg)` | `#0d1117` | Page background, full-screen containers |
| Secondary (30%) | `var(--bg2)` / `var(--bg3)` | `#161b22` / `#1c2128` | Sticky filter bar (`--bg2`), list item cards (`--bg3`), form inputs (`--bg3`) |
| Accent (10%) | `var(--acl)` | `#3b82f6` | Active tab underline border, primary action buttons (직원 추가, 개소 추가), active tab text |

Accent reserved for:
1. Active tab underline (`borderBottom: 2px solid var(--acl)`)

3. Active tab text color
4. Form submit / save button background

- `var(--safe)` `#22c55e` — active/enabled state indicator dot
- `var(--t3)` `#6e7681` — inactive/disabled staff or checkpoint rows (dimmed text)
- `var(--warn)` `#f59e0b` — password-reset confirmation banner background tint

## Interaction Contracts

### Tab Switching

- No page navigation. `useState<AdminTab>` internal to `AdminPage`.
- Switching clears the other tab's filter state (category dropdown resets to "전체").
- No animation between tabs (instant content swap).

### Staff List Loading State

- Skeleton cards: 3 placeholder rows, `background: var(--bg3)`, border-radius 12px, height 64px, `animation: blink 2s ease-in-out infinite`.

### Checkpoint List Loading State

- Same skeleton as Staff. Show after category dropdown is selected and data is loading.

### Form Validation

- Required fields (name, staffId for staff; name, category for checkpoint): show red border `border-color: var(--danger)` on blur if empty. No inline error text under field — toast only.
- Staff ID (사번): 10-digit numeric, `inputmode="numeric"`. If non-numeric or wrong length: toast "사번은 10자리 숫자여야 합니다".
- Save button disabled state when required fields are empty: `opacity: 0.4`, `cursor: not-allowed`.

### Toast Notifications

All via `react-hot-toast`. Korean copy (see Copywriting Contract).

- Success: default (white/dark auto-theme follows system, already handled by react-hot-toast defaults)
- Error: `toast.error(message)`
