# Phase 24 — UI Design Contract: 소화기 자산-위치 분리

> Visual and interaction contract for the extinguisher asset/location split phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Primacy:** This phase REUSES the established `cha-bio-safety` dark-theme token system (`var(--bg)`, `var(--t1)`, `var(--acl)`, `var(--danger)` …) and inline-style + Tailwind utility class hybrid. **Do not introduce new tokens, gradients, or component libraries.** Every new visual element must be expressible with the tokens listed below.
>
> **Memory rules in force for this phase:**
> - 「디자인 변경 전 상의 필수」 — every layout/structural change in this contract must be sketched (HTML 시안) and approved BEFORE inline application during execution.
> - 「디자인 조정은 시안 먼저」 — spacing/sizing in particular requires sketch HTML approval. The values declared below are the contract; sketch confirms how they compose.
> - 「BottomNav 하단 갭 구현 방식」 — Mobile pages with bottom nav must reserve `paddingBottom` (not absolute bottom offsets) for the BottomNav + safe area.

## Spacing Scale

The project's de-facto spacing pattern (from RemediationPage, LegalPage, InspectionPage, FloorPlanPage scout) uses a 2px base step (multiples of 2 inside ≤16, multiples of 4 above). For Phase 24 we lock to a multiples-of-4 contract with two named exceptions that already exist in the codebase.

| Token | Value | Usage |
|-------|-------|-------|
| xxs | 4px | Icon-to-text gap inside chips/badges; row inner padding |
| xs | 8px | Card inner gaps; button-row gap; filter chip gap |
| sm | 12px | Card padding (carryover from RemediationPage `padding:12`) |
| md | 16px | Modal content padding; section vertical padding; page horizontal padding |
| lg | 20px | Modal wrapper padding (carryover from FloorPlanPage marker-edit modal `padding:20`) |
| xl | 24px | Section breaks; legend `paddingBottom:26` (existing) approximates this |
| 2xl | 32px | Page-top spacing on desktop master-detail layouts |
| 3xl | 48px | Empty-state vertical padding (`padding:'40px 16px'` existing — round to 48 for new screens) |

**Exceptions (inherited from codebase, do not "fix"):**
- `padding:'40px 16px'` for empty states — keep when matching existing screens (Remediation pattern), use 48px only on new isolated screens.
- `padding:'1px 12px 26px'` on FloorPlan legend — locked by recent fix commit `4f6b8c0`. **Do not change**. New legend additions must respect existing `minHeight: 100` and `paddingBottom: 26`.
- `padding:'2px 6px'` / `padding:'2px 8px'` for status badges — keep as-is to match LegalPage finding-status badge.
- Touch targets on the new `ExtinguishersListPage` cards: minimum tap height **44px** for action buttons (per iOS HIG). Use `height: 36` only inside dense filter rows where the existing `LegalPage` filter bar uses `height: 28~36`.

**BottomNav reservation rule (memory rule 「BottomNav 하단 갭 구현 방식」):**
- Pages that show BottomNav (`/extinguishers` will, since it is in `SideMenu` not `MOBILE_NO_NAV_PATHS`): scrollable content container must end with `paddingBottom: 'calc(var(--sab) + 70px)'` to clear BottomNav (54px) plus breathing room. **Never** use absolute `bottom` offsets to push content above the nav.

## Typography

The codebase uses 10–18 px sizes pervasively. Phase 24 LOCKS this to 4 sizes + 2 weights for the 3 new/modified surfaces (ExtinguishersListPage, marker-modal updates, InspectionPage card additions).

| Role | Size | Weight | Line Height | Where used |
|------|------|--------|-------------|-----------|
| Caption / Hint | 11px | 500 (medium) | 1.4 | Filter labels, mapping-state badge text, change-counter `변경: 2 / 3`, banner microcopy, `var(--t2)`/`var(--t3)` |
| Body / Field value | 13px | 400 (regular) | 1.5 | Form input text, card field values, modal body copy, `var(--t1)`/`var(--t2)` |
| Label / Button / Card title | 14px | 700 (bold) | 1.3 | Card titles (소화기 종류 + 증지번호), primary/secondary buttons, modal field labels, `var(--t1)` |
| Heading / Modal title | 16px | 700 (bold) | 1.25 | Page header (`소화기 관리`), modal headings (`정보 수정`, `소화기 배치`), section titles |

**Color × text mapping (existing convention):**
- Primary value text → `var(--t1)`
- Secondary / metadata → `var(--t2)`
- Tertiary / placeholder → `var(--t3)`
- Field labels above inputs → 11px / weight 500 / `var(--t3)` (matches FloorPlanPage marker modal `구역` label).

**Number / serial monospace:**
- 증지번호 / 제조번호 / 형식승인 should render in `font-family: 'JetBrains Mono', monospace` at 13px when displayed in card detail (improves visual scan of serial digits). Korean 종류/제조업체 stays in Noto Sans KR.

**Forbidden in this phase:**
- Sizes outside {11, 13, 14, 16}.
- Weights outside {400, 700}. (No 500 / 600 for new components beyond the explicit Caption row; the existing screen migrations may keep 500/600 carryover but new ExtinguishersListPage components must use only 400 or 700.)
- Custom letter-spacing — keep default.

1. **빈 마커 (미배치) 시각** — fill `var(--danger)` `#ef4444`, white inner `❓` glyph at 60% of marker size, no stroke. Locked: see §"빈 마커 spec" below.

5. Disabled-action red border on the saving CTA when validation fails (counter > 3): inline border `1px solid var(--danger)55` plus `cursor:'not-allowed'`.
6. Legend `미배치` row item icon (the same red ❓ at 13px to match other legend marker icons).

Badge geometry: `fontSize: 11`, `fontWeight: 700`, `padding: '2px 8px'`, `borderRadius: 5` — identical to `LegalPage` finding-status badge.

### 빈 마커 spec (LOCKED)

- **Fill:** solid `var(--danger)` `#ef4444` (no transparency on the disk itself).
- **Shape:** circle (matches existing `MarkerIcon` disk geometry — same `size` prop as other markers, default `size: 20` on canvas, `size: 13` in legend).
- **Glyph:** white `❓` rendered as overlay `<text>` (or absolutely-positioned `<span>`) at 60% of marker height, font weight 700, `text-shadow: 0 1px 2px rgba(0,0,0,0.45)` for legibility on the red disk. No outer stroke.
- **Z-order:** Same layer as other markers — clicking it MUST trigger the inspection-mode "소화기 미배치" guidance modal (not silent).
- **Legend row 2 entry:** `❓ 13px var(--danger) disk + white text` followed by 11px `var(--t2)` label `미배치`. Place AFTER the `resolved` 완료 entry so existing order (정상→주의→불량→완료→미배치) reads as a state continuum.
- **Re-flow guard:** The legend already has `minHeight: 100` and `padding: '1px 12px 26px'` (commits `c7d5531`, `4f6b8c0`) — adding `미배치` must not overflow on iPhone SE (375 px). Confirmed safe via `flex-wrap: wrap` on `rowStyle` and the existing `gap: '6px 8px'`.

## Interaction Specs

### Focus / pressed / disabled visual states

- **Hover (desktop only):** Primary buttons brighten by 10% (`filter: brightness(1.1)`). Cards add `border-color: var(--bd2)` and lift via `transform: translateY(-1px)` with `transition: transform .12s ease`.
- **Pressed (touch):** No `:active` color change required — rely on existing `WebkitTapHighlightColor: transparent` + immediate state feedback (modal opens, toast appears).
- **Focus (keyboard):** Inputs get `border-color: var(--acl)`. Buttons get `outline: 2px solid var(--acl); outline-offset: 2px`. Do not remove default outline without replacement.
- **Disabled (counter > 3, saving):** `background: var(--bd2)`, `color: var(--t3)`, `cursor: not-allowed`. NEVER hide the button.

### Loading / saving feedback

- During registration / edit / assign / swap / dispose / delete: Save button shows spinner (existing 14px spinner from RemediationPage detail: `width:14, height:14, border:'2px solid var(--bd2)', borderTopColor:'#fff', borderRadius:'50%', animation:'spin .7s linear infinite'`) replacing label text. Modal stays open until success/error.
- On success: close modal, refetch list query, toast per §Copywriting, query string `?fromMarker=` cleared on auto-mapping success.

### Animation

- Modal entrance: existing `slideUp` keyframe (defined in tailwind.config.js, used pervasively): `animation: 'slideUp 0.28s ease-out both'`. Backdrop fade-in (180ms).
- Card expansion (collapsed → expanded): height transition only via `transition: 'all .18s ease'` on the wrapper. No separate enter/exit animation for the action row — it appears instantly inside the expanded card.
- ❓ marker on map: no animation by default. (Existing markers use no entrance animation.)
