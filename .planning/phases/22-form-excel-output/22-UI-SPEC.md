## Component Inventory

### 1. Month Navigator Header

Mirrors `DailyReportPage` `dateNav` exactly, substituting date with `YYYY년 MM월` format.

```
[ ‹ ]  [  2026년 4월  ]  [ › ]
```

- Left arrow (`‹`): always enabled; navigates to previous month
- Right arrow (`›`): always enabled (no future-month restriction — unlike daily which caps at today)
- Center label: 90px minimum width, `fontSize: 13, fontWeight: 700, color: var(--t1)`, text-align: center
- Arrow buttons: `navBtn` style — 28×28px, `borderRadius: 7`, `background: var(--bg3)`, `border: 1px solid var(--bd)`
- On click of center label: open native `<input type="month">` picker (positioned absolutely below label)

**Source:** `CONTEXT.md` D-06; DailyReportPage `dateNav` block (line 388-401).

### 2. Section Card

Reuses `DailyReportPage` `card` style constant verbatim:
```js
{ background: 'var(--bg2)', borderRadius: 14, border: '1px solid var(--bd)', padding: 14, marginBottom: 10 }
```

Card heading: `fontSize: 13, fontWeight: 700, color: var(--t1), marginBottom: 8`

### 3. Textarea Input

Reuses `DailyReportPage` `textareaStyle` constant verbatim:
```js
{
  width: '100%', boxSizing: 'border-box',
  background: 'var(--bg3)', border: '1px solid var(--bd)', borderRadius: 9,
  color: 'var(--t1)', fontSize: 12, fontFamily: 'inherit', fontWeight: 400,
  padding: '10px 12px', resize: 'vertical', outline: 'none', lineHeight: 1.6,
}
```

On focus: add `border: '1px solid var(--bd2)'` (no custom outline — matches project pattern).

Rows:
- 소방시설 확인내용 (fire_content): `rows={4}`, read-prefilled static text, editable by admin only
- 피난방화시설 확인내용 (escape_content): `rows={3}`, same
- 화기취급감독 확인내용 (gas_content): `rows={2}`, same
- 기타사항 확인내용 (etc_content): `rows={3}`, dynamic prefill
- 소방시설 조치내역 (fire_action): `rows={3}`, conditionally highlighted when 불량
- 피난방화시설 조치내역 (escape_action): `rows={3}`, same

Read-only non-admin: add `readOnly`, change `background: 'var(--bg2)'`, `color: 'var(--t2)'`, `cursor: 'default'`.

### 4. 양호/불량 Toggle

Button-pair pattern (NOT a switch or radio). Two inline buttons inside a card row.

```
[ 양호 ]  [ 불량 ]
```

- Container: `display: flex, gap: 6`
- Button size: `padding: '5px 16px'`, `borderRadius: 7`, `fontSize: 12`, `fontWeight: 700`
- Inactive state: `background: var(--bg3)`, `border: '1px solid var(--bd)'`, `color: var(--t2)`
- 양호 active: `background: var(--safe)` (#22c55e), `border: '1px solid var(--safe)'`, `color: #fff`
- 불량 active: `background: var(--danger)` (#ef4444), `border: '1px solid var(--danger)'`, `color: #fff`
- Disabled (non-admin): both buttons have `cursor: default`, `opacity: 0.5`

When `fire_result === 'bad'`: the 조치내역 textarea label gets `color: var(--warn)` to draw attention.

**Source:** `CONTEXT.md` D-08 Claude's Discretion — button-pair chosen over radio/switch for mobile tap friendliness and visual clarity.

### 5. 관리자 이름 Text Input

Single-line `<input type="text">` in the 기본 정보 card.

Style:
```js
{
  width: '100%', boxSizing: 'border-box',
  background: 'var(--bg3)', border: '1px solid var(--bd)', borderRadius: 9,
  color: 'var(--t1)', fontSize: 13, fontFamily: 'inherit', fontWeight: 400,
  padding: '9px 12px', outline: 'none',
}
```

On focus: `border: '1px solid var(--bd2)'`.
Non-admin: `readOnly`, `background: var(--bg2)`, `color: var(--t2)`.

### 6. "수정됨" Dirty Indicator

Inline text span, placed immediately to the right of the save button label.

```
[  저장  · 수정됨  ]
```

Implementation: rendered inside the save button as a trailing `<span>`:
```jsx
<span style={{ fontSize: 11, fontWeight: 400, color: 'var(--warn)', marginLeft: 6 }}>· 수정됨</span>
```

Visible when: current form values differ from values loaded from server.
Hidden when: values match loaded state OR after successful save.

**Source:** `CONTEXT.md` D-11.

### 7. Fixed Footer Buttons

Two buttons in a row, pinned to viewport bottom.

Container:
```js
{
  position: 'fixed', bottom: 0, left: 0, right: 0,
  padding: '10px 16px',
  paddingBottom: 'calc(10px + var(--sab))',  // iOS safe-area
  background: 'var(--bg2)',
  borderTop: '1px solid var(--bd)',
  display: 'flex', gap: 8, zIndex: 10,
}
```

- **저장 버튼** (primary): `flex: 1`, `padding: 11`, `borderRadius: 9`, `border: 'none'`
  - Normal (admin): `background: 'linear-gradient(135deg,#1d4ed8,#2563eb)'`, `color: '#fff'`, `fontSize: 13, fontWeight: 700`
  - Saving (mutation in flight): `background: var(--bg3)`, `color: var(--t3)`, `cursor: default` — label becomes `'저장 중...'`
  - Disabled (non-admin): `background: var(--bg3)`, `color: var(--t3)`, `cursor: default`

- **엑셀 출력 버튼** (secondary): `flex: 1`, `padding: 11`, `borderRadius: 9`, `border: '1px solid var(--bd)'`
  - Normal (admin): `background: var(--bg2)`, `color: var(--t1)`, `fontSize: 13, fontWeight: 700`
  - Generating: `background: var(--bg3)`, `color: var(--t3)`, `cursor: default` — label becomes `'출력 중...'`
  - Unsaved changes present: show confirmation dialog before generating (see Copywriting section)
  - Disabled (non-admin): `background: var(--bg3)`, `color: var(--t3)`, `cursor: default`

**Source:** `CONTEXT.md` D-09; DailyReportPage download button styles (lines 328-377).

### 8. Loading Skeleton

When query is fetching: replace each textarea with an animated skeleton block.
```js
{ background: 'var(--bg4)', borderRadius: 4, height: 12, width: '70%', animation: 'blink 2s ease-in-out infinite' }
```
Pattern matches DailyReportPage line 302.

### 9. Error State Block

When query errors: render inside the card body:
```jsx
<div style={{ fontSize: 11, color: 'var(--danger)' }}>데이터 불러오기 실패 — 다시 시도해 주세요</div>
```

### 10. Desktop Layout

Mirrors DailyReportPage desktop layout: single-panel scrollable form (no A4 print preview panel — worklog is Excel-only output, no print preview required).

```
+-------------------------------------------+
|  DesktopSidebar  |  WorkLog form content   |
|                  |  (scrollable, flex:1)   |
+------------------+-------------------------+
```

Desktop padding: `padding: '24px 32px'`.
Form max-width: none (fills available space).

## Screen States

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Loading | Initial fetch / month change | Skeleton blocks in each textarea area (blink animation) |
| Prefilled (new month) | No saved record → preview API response | All fields populated, dirty indicator hidden, footer buttons active |
| Loaded (existing month) | Saved record found | All fields populated from DB, dirty indicator hidden |
| Dirty | Any field edited | "· 수정됨" appears in save button, no auto-save |
| Saving | Save button tapped | Button label → "저장 중...", button disabled, all form inputs remain editable |
| Saved | Mutation success | Dirty indicator disappears, toast success shown |
| Generating Excel | 엑셀 출력 tapped (no unsaved changes) | Button label → "출력 중...", button disabled |
| Unsaved + Export | 엑셀 출력 tapped with unsaved changes | Confirmation dialog appears (see Copywriting) |
| Error | API failure | Error text inside affected card, toast error |
| Non-admin | staff.role !== 'admin' | All inputs read-only, save/export buttons disabled |

## Interaction Flows

### Flow 1: Initial Page Load
1. Page renders with skeleton blocks in all textarea areas
2. `workLogApi.get(ym)` fires
3a. Record found → populate all fields from saved data; dirty indicator hidden
3b. No record → `workLogApi.preview(ym)` fires → populate from auto-aggregate; show empty state note briefly
4. Skeleton replaced by populated fields

### Flow 2: Month Change
1. User taps `‹` or `›`
2. If dirty: no prompt (no auto-save, CONTEXT D-10) — user navigates away; unsaved changes silently discarded
   - Consider: add "수정됨" in month nav label to remind user before navigating, but no blocking prompt (low friction, monthly cadence)
3. Month state updates, fetch cycle restarts (Flow 1)

### Flow 3: Save
1. User taps `저장`
2. Button becomes `저장 중...`, disabled
3. `workLogApi.save(ym, payload)` fires (PUT upsert)
4. Success: toast `저장되었습니다`, dirty indicator clears, loaded state refreshes
5. Failure: toast `저장 실패`, button re-enables

### Flow 4: Excel Export (clean state)
1. User taps `엑셀 출력`
2. Button becomes `출력 중...`, disabled
3. `generateWorkLogExcel(ym, currentFormValues)` runs client-side
4. Browser triggers download: `소방안전관리자_업무수행기록표_${year}년_${month}월.xlsx`
5. Button re-enables

### Flow 5: Excel Export (dirty state)
1. User taps `엑셀 출력` with unsaved changes
2. Confirmation dialog appears: "저장되지 않은 변경사항이 있습니다 / 저장 후 엑셀을 출력하시겠습니까?" with buttons `저장 후 출력` / `취소`
3a. `저장 후 출력`: execute Flow 3 then Flow 4 in sequence
3b. `취소`: dismiss dialog, no action

### Flow 6: Non-admin View
1. All textareas and inputs are `readOnly`
2. Save and 엑셀 출력 buttons are `disabled` with `opacity: 0.5`
3. On tap/hover of disabled buttons: tooltip `관리자만 저장할 수 있습니다`
4. Read flow (Flow 1) proceeds normally for preview
