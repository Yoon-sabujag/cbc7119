## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap in PhotoButton dashed cell (`gap: 4`) |
| sm | 8px | Spacing between thumbnail slots in PhotoGrid row |
| md | 16px | Section padding inside detail page content areas |
| lg | 24px | Vertical gap between label and PhotoGrid row |
| xl | 32px | Padding between form sections in BottomSheet |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- **72px thumbnail size** — fixed square dimension for `PhotoGrid` thumbnails (established by existing `PhotoButton`, CONTEXT.md D-03). Not a spacing token; it is a component dimension.
- **44px minimum touch target** — the add-slot `+` button inside `PhotoGrid` must have a minimum tappable area of 44px even though the visual cell is 72px (already satisfied by 72px).
- **20px remove badge** — `×` removal button on thumbnail top-right corner (established by `PhotoButton` pattern, `width: 20, height: 20`).

Source: `src/components/PhotoButton.tsx` (existing), CONTEXT.md D-03, CONTEXT.md D-04

## Interaction States

### PhotoGrid — Add-slot button

| State | Appearance |
|-------|-----------|
| Default (can add) | 72×72 `var(--bg2)` background, `1px dashed var(--bd2)` border, 📷 emoji at 22px, "사진 첨부" label at 11px `var(--t3)` |
| Hidden (5 photos) | `display: 'none'` — add button not rendered when `slots.length >= 5` |
| Disabled (uploading) | Pointer events disabled while `isUploading === true` — no visual change specified; executor may grey label |

### PhotoGrid — Thumbnail

| State | Appearance |
|-------|-----------|
| Default | 72×72 image, `borderRadius: 10`, `border: '1px solid var(--bd)'`, cursor pointer |
| Uploading | Same image + `rgba(0,0,0,0.4)` overlay + "업로드 중" centered text |
| Error | Same image + red tint not required; error caption below thumbnail at `color: 'var(--danger)'` |
| Remove badge | Always visible when in upload mode — `×` at top-right |

### Lightbox

| State | Behavior |
|-------|---------|
| Closed | Not rendered |
| Open | Full-screen overlay; touch/swipe to navigate between photos; close on backdrop tap or close button |

## Copywriting Contract

All copy in Korean. Concise, action-oriented.

| Element | Copy |
|---------|------|
| Primary CTA | "사진 첨부" (add-slot button label) |
| Add-slot emoji | 📷 (camera emoji — decorative, established by PhotoButton) |
| Upload progress | "업로드 중" (shown as overlay on uploading thumbnail) |
| Upload error (per slot) | "업로드 실패" (error caption below failed thumbnail) |
| Upload error retry hint | Not required in Phase 12 — retry by removing and re-adding the photo |
| Empty state (no photos) | Single add-slot button shown; no separate empty state message needed — button is self-explanatory |
| Limit reached (5 photos) | Add-slot button hidden — no message needed (button absence communicates limit) |
| Remove button aria-label | `aria-label="사진 제거"` (matches existing PhotoButton pattern) |
| Section label — finding photos | "지적 사진" |
| Section label — resolution photos | "조치 사진" |

Destructive actions in this phase:
- **Photo removal:** Tap `×` badge on thumbnail. No confirmation dialog — removal is low-stakes (photo not yet saved to server if in upload mode; in display mode this phase does not expose deletion). Immediate removal with blob URL revocation.

## Anti-Patterns (Executor Must Avoid)

| Banned | Required Instead |
|--------|-----------------|
| `capture="environment"` on multi-photo input | No `capture` attribute — `accept="image/*"` only (CONTEXT.md D-07) |
| `COALESCE(?, photo_keys)` in SQL UPDATE | Full replacement: `photo_keys = ?` with `JSON.stringify(array)` (RESEARCH.md Pitfall 2) |
| Sequential `await fetch` per photo in loop | `Promise.allSettled` for parallel uploads (RESEARCH.md Pitfall, Standard Stack) |
| Hardcoded hex colors in component code | CSS variable references: `var(--bg2)`, `var(--danger)`, `var(--bd2)`, `var(--t3)` |
| `<a download>` for any download in this phase | Not applicable to Phase 12; no download UI this phase |
| Tailwind utility classes in component files | Inline style objects with CSS variable references (CLAUDE.md + RESEARCH.md constraint) |
| Missing blob URL revocation on unmount | `useEffect` cleanup must revoke all slot preview URLs and abort controllers |
