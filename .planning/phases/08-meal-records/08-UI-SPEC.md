## Interaction Contract

### Tap Cycle (primary interaction)

| State | tap() result | Visual |
|-------|-------------|--------|
| providedMeals=0 | no-op | cell opacity 0.4 |
| skipped=0, provided=1 | skippedвҶ’1 | в‘  badge appears |
| skipped=1, provided=1 | skippedвҶ’0 | badge disappears |
| skipped=0, provided=2 | skippedвҶ’1 | в‘  badge appears |
| skipped=1, provided=2 | skippedвҶ’2 | в‘Ў badge appears |
| skipped=2, provided=2 | skippedвҶ’0 | badge disappears |

**Optimistic UI:** Update local state immediately before API call. Roll back on API error with toast.error.

**Debounce:** No debounce. Each tap is a discrete state machine step. If user taps rapidly, each tap enqueues an upsert call. Duplicate API calls for the same date are safe (INSERT OR REPLACE).

### Month Navigation

- Pressing вҖ№ decrements month; pressing вҖә increments month
- Re-fetch meal records and leaves for new month on navigate
- React Query key: `['meal', year, month]` and `['leaves', year, monthStr]`

### Loading State

- Calendar area: centered circular spinner (`border: 2px solid var(--bd2)`, `borderTopColor: var(--acl)`, 24px, spin animation)
- Summary cards: skeleton loaders (`background: var(--bg3)`, `borderRadius: 12`, `height: 64px`, `animation: blink 2s ease-in-out infinite`)
- No loading state on individual cell taps (optimistic update is instant)

## Implementation Notes for Executor

1. **No new npm packages.** All dependencies (React, date-fns, react-query, shiftCalc, leaveApi, react-hot-toast) already installed.

2. **CSS variable usage is mandatory.** Never hardcode `#0d1117` or other token hex values directly in inline styles. Always use `var(--bg)` etc.

3. **Inline style objects only.** No Tailwind utility classes on components. Tailwind is used for layout containers only if needed. Match the project's inline-style convention.

4. **Calendar cell tap handler must check providedMeals > 0** before advancing cycle. Return early (optionally with toast) if the day has no provision.

5. **Skip badge content:** Use Unicode circled numbers в‘  (U+2460) and в‘Ў (U+2461) inside a small blue badge positioned at `top: 2, right: 2` relative to the cell. Badge is 12px font with `padding: 0 3px`, `minWidth: 14px`. If circled numbers cause rendering issues on Android WebView, fall back to plain "1" / "2" with the blue pill background.

6. **Summary card layout:** 4 cards in flex row with `overflowX: auto` on the container. On narrow devices all 4 must be visible without forcing scroll вҖ” target `minWidth: 72px` per card so they fit in 320px width. On wider screens cards expand equally.

7. **Month navigation state:** Use `useState<[number, number]>` for `[year, month]` (0-indexed month). When month nav triggers, invalidate both `['meal', year, month]` and `['leaves', year, monthStr]` React Query keys.

8. **`skipped_meals` upper bound** must be enforced in the tap handler using the calculated `providedMeals` value, not a hardcoded value. The cycle is: `next = (current + 1) > providedMeals ? 0 : current + 1`.

9. **SideMenu activation:** Change `soon: true` вҶ’ `soon: false` for the мӢқлӢ№ л©”лүҙ item. The label "мӢқлӢ№ л©”лүҙ" can remain as-is or be updated to "мӢқмӮ¬ кё°лЎқ" вҖ” executor's discretion (Claude's Discretion per CONTEXT.md).

10. **Route:** `/meal` is already in `NO_NAV_PATHS` in App.tsx (CONTEXT.md canonical ref line 81). Only a `<Route path="/meal" element={<MealPage />} />` lazy import addition is needed.
