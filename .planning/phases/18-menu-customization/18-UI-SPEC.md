## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)` = `#0d1117` | Page background, overlay base |
| Secondary (30%) | `var(--bg2)` = `#161b22`, `var(--bg3)` = `#1c2128` | Panel surface, Row background, edit card background |
| Accent (10%) | `var(--acl)` = `#3b82f6` | Toggle ON state, primary save button, active QR button gradient |
| Destructive | `var(--danger)` = `#ef4444` | No destructive actions in Phase 18 — not used |

Accent reserved for:
1. Toggle switch background when `on === true` — exactly as in the existing `Toggle` primitive (`SettingsPanel.tsx:24`)
2. "설정 저장" primary button background — matches existing SideMenu editMode save button pattern (`SideMenu.tsx:240`)
3. QR center FAB gradient remains `linear-gradient(135deg,#1d4ed8,#0ea5e9)` — do not recolor

Accent is NOT used for: row backgrounds, borders, section labels, inactive states, arrows/chevrons, or any read-only display element.

Border tokens:
- `var(--bd)` = `rgba(255,255,255,0.07)` — default dividers, card borders in edit mode
- `var(--bd2)` = `rgba(255,255,255,0.13)` — cancel button border, dashed "메뉴 편집" button border

Text tokens:
- `var(--t1)` = `#e6edf3` — primary text (row labels, headings)
- `var(--t2)` = `#8b949e` — secondary text (cancel button label, close button icon)
- `var(--t3)` = `#6e7681` — tertiary text (section labels, sub-labels, sub-text, arrow buttons in disabled state)

## Component Inventory

These existing primitives MUST be reused. Do not re-create them.

### `Row` (SettingsPanel.tsx:54–64)
- Props: `label: string`, `sub?: string`, `children?: React.ReactNode`, `onClick?: () => void`
- Renders: `padding: '10px 12px'`, `background: 'var(--bg3)'`, `borderRadius: 9`, `marginBottom: 5`
- Right slot: any children (Toggle, chevron, badge)
- Use for: every SideMenu item visibility row, every BottomNav item row in the settings section

### `Toggle` (SettingsPanel.tsx:15–35)
- Props: `on: boolean`, `onChange?: (v: boolean) => void`, `disabled?: boolean`
- Renders: `width:38, height:21`, track `#2563eb` when on / `var(--bg4)` when off, thumb 17×17 white circle
- Note: The existing SideMenu edit toggle is 44×24 — do NOT use it. Use the SettingsPanel `Toggle` (38×21) for new menu settings rows for visual consistency with the rest of the settings page.

### Section header pattern (SettingsPanel.tsx:312)
```
fontSize: 9, fontWeight: 700, color: 'var(--t3)', letterSpacing: '.08em',
textTransform: 'uppercase', marginBottom: 6
```
Use this exact style for "메뉴 설정", "하단 내비게이션", "사이드 메뉴" sub-section labels.

### Up/Down arrow reorder button (SideMenu.tsx:213–216)

- Disabled (at boundary): `opacity:0.35`, `cursor:'default'`, `pointerEvents:'none'`

## Layout: Menu Settings Section in SettingsPanel

The new "메뉴 설정" section is added inside `SettingsPanel.tsx` below the "화면" section and above the "계정" section.

### Section structure

```
[Section header] 메뉴 설정

[Sub-header] 하단 내비게이션

[Row — 대시보드] visible toggle + up/down arrows
[Row — 점검] visible toggle + up/down arrows
[Row — QR 스캔] locked badge, no arrows (center-fixed)
[Row — 조치] visible toggle + up/down arrows
[Row — 승강기] visible toggle + up/down arrows

[Sub-header] 사이드 메뉴

[Per section accordion]:
  [Section title row] drag handle or arrows + trash (if empty) + chevron expand
  [Per item row] visible toggle + up/down arrows + section move button

[Row — 새 섹션 추가] dashed border button

[Footer bar] 취소 | 설정 저장
```

The section renders inline within the settings page scroll — no modal overlay. This matches the inlined `ChangePasswordForm` precedent in `SettingsPanel.tsx:369`.

## Interaction Contract

### BottomNav reorder

- 4 reorderable items: 대시보드, 점검, 조치, 승강기
- QR is center-fixed: row renders with lock icon and label "중앙 고정" in `var(--t3)`, no toggle, no arrows
- Up/down arrow buttons swap adjacent items in local state (matches existing SideMenu editMode pattern exactly — `SideMenu.tsx:213`)
- Each item row also has a visible Toggle on the right (reuse `Toggle` primitive)
- Rows use existing Row primitive with left slot = arrow pair, right slot = Toggle

### SideMenu section editor

- Sections render as expandable accordion rows (collapsed by default)
- Expanded section shows its items with: up/down arrows (left) + label (center) + section-move chevron + visible Toggle (right)
- Section row itself has: up/down section-order arrows + section title (editable inline on tap) + item count badge + expand chevron
- "새 섹션 추가" at bottom: dashed-border button, same style as existing "⚙ 메뉴 편집" button (`SideMenu.tsx:195` — `border:'1px dashed var(--bd2)'`, `background:'transparent'`, `color:'var(--t3)'`, `fontSize:10`, `fontWeight:600`)
- Tapping "새 섹션 추가" inserts an inline text input immediately above the button (no modal)
- Empty section shows trash icon button inline on section row; tapping shows confirmation text inline ("비어 있는 섹션입니다. 삭제하시겠습니까? [삭제] [취소]") — no modal overlay
- Item section-move: tap a "섹션 이동" icon on the item row to open a compact inline picker (a small list of section names rendered as buttons in a bg2 card below the row, z-index elevated, dismissed by tapping elsewhere)

### Save / cancel flow

- Changes accumulate in local draft state (not auto-saved)
- "설정 저장" button: full-width at section bottom, `background:'var(--acl)'`, `color:'#fff'`, `height:40`, `borderRadius:8`, `fontSize:12`, `fontWeight:700`
- "취소" button: adjacent left, `flex:1` to save's `flex:2`, `border:'1px solid var(--bd2)'`, `background:'var(--bg)'`, `color:'var(--t2)'`
- Save calls `settingsApi.saveMenu(newConfig)` then `queryClient.invalidateQueries(['menu-config'])`
- Cancel resets draft state to server-fetched config without API call
- While saving: save button shows "저장 중...", `opacity:0.5`, pointer-events none

### Read-only states

- BottomNav renders live from saved config (useMemo + menuConfig, matching `appliedMenu` pattern in `SideMenu.tsx:60`)
- SideMenu strips its own editMode completely (D-03, D-05 from CONTEXT.md) — it becomes read-only filtered by saved config

Note: "취소" is a single-word label. This is intentional Korean UX convention — "취소" (cancel) is universally understood as a standalone action label in Korean-language interfaces and requires no additional context word.

No destructive-action confirmation dialogs in this phase except empty-section delete, which uses inline confirmation text (not a modal).

## Accessibility & Touch Targets

- All tap targets minimum 44px tall (Row primitive height via `padding:'10px 12px'` + 12px font = ~36px rendered height — executor must add `minHeight:44` to Row wrapper for touch targets when used in menu settings)
- Arrow buttons: wrap in a `div` with `width:28, height:44` to meet 44px vertical touch target while keeping the visual 14px character
- Toggle component already meets 38×21 — acceptable for horizontal (thumb travel); vertical touch area inherited from Row's 44px minHeight
- QR lock row must have `aria-label="QR 스캔 — 중앙 고정, 변경 불가"` on the row container
