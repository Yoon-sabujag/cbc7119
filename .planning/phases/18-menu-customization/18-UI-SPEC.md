## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline icon spacing |
| sm | 8px | Gap between action buttons (up/down arrows, trash), input gap within form |
| md | 12px | Row horizontal padding (matches existing Row: `padding: '10px 12px'`) |
| lg | 16px | Section-to-section gap within SettingsPanel, entry list gap |
| xl | 24px | Section header top margin |
| 2xl | 32px | — |
| 3xl | 48px | — |

Exceptions (Phase 18 owned):
- Touch targets for arrow buttons: minimum 32×32px (mobile tap safety; not 44px because arrows appear in a dense list)
- Divider title input height: 40px (matches existing ChangePasswordForm inputs)
- Save button height: 40px (matches existing button height convention)

Inherited spacing (not owned or changed by this phase — executor must preserve as-is):
- Row primitive top/bottom padding: 10px (defined in SettingsPanel.tsx Row component — do not modify)
- Section label bottom margin: 6px (defined by existing SettingsPanel section heading convention — do not modify)

These inherited values are not on the 4-point scale but must not be changed because they belong to shared primitives outside Phase 18 scope.

Source: Extracted directly from SettingsPanel.tsx Row label (12px), sub (10px), section uppercase heading (9px/700), and input fields (13px).

Notes:
- All Korean text. No display-size heading (28px+) appears in the settings panel.
- Divider title in the editor list: rendered at 12px/400 when read-only, switches to a 13px/400 input on tap (matches the ChangePasswordForm input pattern).

- Two weights only: 400 (body, sub, input, row labels) and 700 (section headings, CTAs, danger confirmations).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)` → #0d1117 | App background |
| Secondary (30%) | `var(--bg2)` → #161b22 / `var(--bg3)` → #1c2128 | SettingsPanel background (`--bg2`), Row cells (`--bg3`) |
| Accent (10%) | `var(--acl)` → #3b82f6 | Active Toggle thumb track, Save button background, focused input border |
| Destructive | `var(--danger)` → #ef4444 | Divider delete trash icon, delete confirmation inline text |

Source: index.css CSS variable definitions, SettingsPanel.tsx inline styles.

**Visual Hierarchy:** Primary visual anchor: '설정 저장' button (full-width, accent background). Entry list is the content area. Add/Reset controls are secondary.

Accent reserved for:
1. Toggle ON state background (`#2563eb` in existing Toggle — maintain this value)
2. "설정 저장" (Save) button background
3. Active/focused state on divider title input border

The accent color must NOT be applied to: arrow buttons, divider rows, item row backgrounds, or section headings.

Secondary surface breakdown:
- SettingsPanel slide-in panel: `var(--bg2)`
- Each Row cell (item row + divider row): `var(--bg3)`
- Disabled / ghost button (Cancel): `var(--bg4)` → #22272e

Divider row visual distinction from item row:
- Divider rows use `var(--bg3)` background same as item rows (they are siblings in the flat list)
- Divider title text: `var(--t2)` → #8b949e at 9px/700 uppercase (mirrors existing section heading style to signal it is a heading marker)
- Left border accent: 2px solid `var(--bd2)` → rgba(255,255,255,0.13) on divider rows to subtly distinguish from item rows without introducing a new color

Hidden item visual state:
- `opacity: 0.4` on item row when `visible: false` (matches the disabled opacity pattern already used for Toggle and buttons in SettingsPanel)
- "숨김" label at 9px in `var(--t3)` shown to the right of the item label when hidden

## Interaction Contract

### MenuSettingsSection — editor list

**Entry order:** Flat array rendered top-to-bottom. Each entry occupies one Row-height cell.

**Move up / Move down arrows:**
- Both arrows always rendered as icon buttons (ChevronUp, ChevronDown from lucide-react, 14px)
- First entry: up arrow `opacity: 0.25`, `pointerEvents: none`
- Last entry: down arrow `opacity: 0.25`, `pointerEvents: none`
- Touch target wrapper: 32×32px centered around the 14px icon

**Item row controls (left to right):**
1. Drag handle area (reserved space 16px wide, no drag behavior in Phase 18 — deferred)
2. Item label (12px/400, `var(--t1)`) — truncated at 1 line with `overflow: hidden; textOverflow: ellipsis`
3. "숨김" badge (9px, `var(--t3)`) — shown only when `visible: false`
4. ChevronUp + ChevronDown (move)
5. Toggle (38×21, ON = visible, OFF = hidden)

**Divider row controls (left to right):**
1. Drag handle area (reserved, 16px)
2. Divider title (9px/700, uppercase, `var(--t2)`) — tapping enters inline edit
3. ChevronUp + ChevronDown (move)
4. Trash icon (Trash2 from lucide-react, 14px, `var(--danger)`)

**Divider title inline edit:**
- Tap on divider title area → replace span with `<input>` (height 40px, matching ChangePasswordForm input style) — autofocus
- Input loses focus (blur) → commit the new title if non-empty; revert to previous value if empty
- Empty title handling: revert silently (no toast) — empty divider titles are not allowed
- Max length: 20 characters

**Divider delete confirmation:**
- Tap trash icon → inline confirmation replaces the row controls:
  `"삭제할까요?"` (10px, `var(--t2)`) + `[취소]` ghost button + `[삭제]` danger button (12px/700)
- Confirmation auto-dismisses after 5 seconds with no action (reverts to normal)
- On confirm: remove entry from draft list immediately (no API call yet — draft state)

**Add divider button:**
- Position: bottom of the entry list, full-width, height 36px
- Style: dashed border `1px dashed var(--bd2)`, background `transparent`, color `var(--t2)`, radius 8
- Label: `+ 구분선 추가` (12px/400)
- Behavior: appends `{ type: 'divider', id: nanoid(), title: '새 구분선' }` to end of draft list; new entry immediately enters inline edit (autofocus)

**Reset to defaults button:**
- Position: below the Add divider button, right-aligned text button (no border/background)
- Label: `기본값으로 초기화` (10px, `var(--t3)`)
- Behavior: tap → inline confirmation `"기본 배치로 되돌릴까요? [취소] [초기화]"` — on confirm, replace draft with default flat list (from D-15 in CONTEXT.md)

**Save button:**
- Position: below Reset button, full-width, height 40px
- Style: `background: var(--acl)`, `color: #fff`, border-radius 9, font 12px/700
- Label: `설정 저장`
- Disabled state: `opacity: 0.4`, `cursor: not-allowed` — when no changes from last saved state
- Loading state: label changes to `저장 중…`, disabled
- On success: toast.success `'메뉴 설정이 저장되었습니다'`, draft syncs to server state
- On error: toast.error `'저장에 실패했습니다. 다시 시도해주세요.'`

### Draft management

- On section mount: initialize draft from `useQuery(['menu-config'])` result (converted to flat array)
- All edits (move, toggle, divider rename, divider delete, divider add) mutate draft only
- Save button commits draft to server via `settingsApi.saveMenu()`
- If panel closes with unsaved changes: no warning — changes are silently discarded (no modal; this is a low-stakes settings panel for a 4-person team)

### SideMenu read-only render (post-Phase 18)

- Flat `SideMenuEntry[]` is iterated in order
- `type: 'divider'` entries render as section headings (existing section heading style: 9px/700/uppercase/`var(--t3)`)
- `type: 'item'` entries with `visible: false` are filtered out entirely (not rendered)
- `type: 'item'` entries with `visible: true` render normally
- Items before the first divider entry (if any) render without a section heading

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | 설정 저장 |
| Section heading in SettingsPanel | 메뉴 설정 |
| Add divider button | + 구분선 추가 |
| Reset button | 기본값으로 초기화 |
| Reset confirmation | 기본 배치로 되돌릴까요? |
| Divider delete confirmation | 삭제할까요? |
| Cancel (inline) | 취소 |
| Confirm delete (inline) | 삭제 |
| Confirm reset (inline) | 초기화 |
| Hidden item badge | 숨김 |
| New divider default title | 새 구분선 |
| Save loading state | 저장 중… |
| Save success toast | 메뉴 설정이 저장되었습니다 |
| Save error toast | 저장에 실패했습니다. 다시 시도해주세요. |
| Empty title revert | (silent — no toast, no message) |

Empty state: The entry list always has content (at minimum the default flat list is present). There is no truly empty state for this list — do not implement an empty state view.

Error state: API save failure → toast.error only (no inline error UI). The draft is preserved so the user can retry.

Destructive actions in this phase:
1. **Divider delete** — inline confirmation within the row (no modal). Auto-dismiss after 5s.
2. **Reset to defaults** — inline confirmation below the list (no modal). Must require explicit tap on "초기화".
