Per D-11: all roles can resolve. Return { success: true }.

**3. next-inspection.ts** — GET /api/elevators/next-inspection:

- Query: SELECT e.id, e.number, e.type, e.install_year, MAX(i.inspect_date) as last_date FROM elevators e LEFT JOIN elevator_inspections i ON i.elevator_id = e.id AND i.type = 'annual' GROUP BY e.id
- Per D-12, D-13, D-15: Calculate cycle using getCycleMonths(type, installYear, today):
  - If install_year IS NOT NULL AND (today.year - install_year >= 25) => 6 months
  - passenger | escalator => 12 months
  - cargo | dumbwaiter => 24 months
- Per D-15: special/detailed inspections also reset the cycle (the query already uses type='annual' which covers all annual records; the inspect_type subtype doesn't matter for cycle reset)
- Compute nextDate = addMonths(lastDate, cycleMonths) using date-fns
- Compute status: overdue if nextDate < today, due_soon if differenceInDays(nextDate, today) <= 30, else ok. If no last_date, status = 'no_record'
- Return array of { elevatorId, elevatorNumber, elevatorType, installYear, lastDate, nextDate, cycleMonths, status, daysUntil }
