**B. Update dashboard/stats.ts:**

Add a query to count elevators with next inspection due within 30 days or overdue. Use the same logic as next-inspection.ts but server-side in the stats handler:

1. Query last annual inspection per elevator:
```sql
SELECT e.id, e.type, e.install_year, MAX(i.inspect_date) as last_date
FROM elevators e
LEFT JOIN elevator_inspections i ON i.elevator_id = e.id AND i.type = 'annual'
GROUP BY e.id
```

2. For each row, compute cycle months (same logic as next-inspection.ts):
   - install_year not null AND age >= 25 => 6
   - passenger | escalator => 12
   - cargo | dumbwaiter => 24

3. Compute next_date = last_date + cycleMonths. If no last_date, count as due.

4. Count elevators where next_date <= today + 30 days OR next_date < today OR no last_date.

5. Use `addMonths` from date-fns (already imported or import it). Keep to 1 additional D1 query (per RESEARCH Pitfall 5).

6. Add `elevInspDueSoon: count` to the stats response object.
