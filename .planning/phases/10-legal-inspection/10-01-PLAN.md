```sql
CREATE TABLE IF NOT EXISTS legal_inspections (
  id               TEXT PRIMARY KEY DEFAULT (lower(hex(randomblob(8)))),
  inspection_type  TEXT NOT NULL CHECK(inspection_type IN ('comprehensive','functional')),
  inspected_at     TEXT NOT NULL,
  agency           TEXT NOT NULL,
  result           TEXT NOT NULL DEFAULT 'pass'
                   CHECK(result IN ('pass','fail','conditional')),
  report_file_key  TEXT,
  memo             TEXT,
  created_by       TEXT NOT NULL,
  created_at       TEXT NOT NULL DEFAULT (datetime('now','+9 hours'))
);

CREATE TABLE IF NOT EXISTS legal_findings (
  id                    TEXT PRIMARY KEY DEFAULT (lower(hex(randomblob(8)))),
  inspection_id         TEXT NOT NULL REFERENCES legal_inspections(id),
  description           TEXT NOT NULL,
  location              TEXT,
  photo_key             TEXT,
  resolution_memo       TEXT,
  resolution_photo_key  TEXT,
  status                TEXT NOT NULL DEFAULT 'open'
                        CHECK(status IN ('open','resolved')),
  resolved_at           TEXT,
  resolved_by           TEXT,
  created_by            TEXT NOT NULL,
  created_at            TEXT NOT NULL DEFAULT (datetime('now','+9 hours'))
);

CREATE INDEX IF NOT EXISTS idx_legal_findings_inspection_id
  ON legal_findings(inspection_id);
CREATE INDEX IF NOT EXISTS idx_legal_findings_status
  ON legal_findings(status);
```

**API 엔드포인트 (5 파일) — education API 패턴 그대로 복사 후 수정:**

1. **functions/api/legal/index.ts** — GET(회차 목록) + POST(회차 생성, admin only per D-13)

GET: LEFT JOIN legal_findings로 finding_count, resolved_count 집계 (N+1 방지, RESEARCH Pitfall 3 참조):
```sql
SELECT li.id, li.inspection_type, li.inspected_at, li.agency, li.result, li.report_file_key, li.memo, li.created_by, li.created_at,
  COUNT(lf.id) AS finding_count,
  SUM(CASE WHEN lf.status='resolved' THEN 1 ELSE 0 END) AS resolved_count
FROM legal_inspections li
LEFT JOIN legal_findings lf ON lf.inspection_id = li.id
GROUP BY li.id
ORDER BY li.inspected_at DESC
```
camelCase 매핑: inspection_type → inspectionType, inspected_at → inspectedAt, report_file_key → reportFileKey, created_by → createdBy, created_at → createdAt, finding_count → findingCount, resolved_count → resolvedCount

POST: role !== 'admin' → 403 "관리자만 가능합니다". Body: { inspection_type, inspected_at, agency, result, report_file_key?, memo? }. INSERT returning id.

2. **functions/api/legal/[id].ts** — GET(회차 상세) + PUT(수정, admin only per D-13)

GET: SELECT * FROM legal_inspections WHERE id = ?. camelCase 매핑 동일.
PUT: role !== 'admin' → 403. COALESCE partial update 패턴 (education/[id].ts 참조).

3. **functions/api/legal/[id]/findings/index.ts** — GET(지적 목록) + POST(지적 등록, all staff per D-12)

GET: SELECT * FROM legal_findings WHERE inspection_id = ? ORDER BY CASE WHEN status='open' THEN 0 ELSE 1 END, created_at DESC. camelCase 매핑: inspection_id → inspectionId, photo_key → photoKey, resolution_memo → resolutionMemo, resolution_photo_key → resolutionPhotoKey, resolved_at → resolvedAt, resolved_by → resolvedBy, created_by → createdBy, created_at → createdAt.
POST: 권한 체크 없음 (전체 직원). Body: { description, location?, photo_key? }. INSERT with inspection_id from params.

4. **functions/api/legal/[id]/findings/[fid].ts** — GET(지적 상세) + PUT(지적 수정, admin only per D-13)

5. **functions/api/legal/[id]/findings/[fid]/resolve.ts** — POST(조치완료, all staff per D-12)
