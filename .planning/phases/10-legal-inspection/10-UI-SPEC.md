| Token | Value | Usage |

| sm | 8px | Card internal row gaps, filter pill gaps |

## Interaction Contracts

### Page 1: /legal — LegalPage (Inspection Round List)

- Self-header (48px): title "법적 점검", no back button on root page — back goes to navigate(-1) which returns to whatever invoked (SideMenu opens it, so -1 is dashboard)
- Header right slot: "+ 점검 등록" button visible only when `role === 'admin'` — 14px, weight 700, color `var(--acl)`
- Card list: vertically scrollable, gap 8px, padding 16px horizontal + 12px vertical
- Cards sorted newest `inspected_at` first
- Tapping a card navigates to `/legal/:id`
- Loading state: 3 skeleton cards (blink animation)
- Empty state (no rounds yet): centered message per copywriting contract
- Error state: centered message per copywriting contract

### Page 2: /legal/:id — LegalFindingsPage (Findings List)

- Self-header (48px): title "지적사항", ← back navigates to `/legal`
- Header right slot: "+ 지적 등록" — visible to all authenticated staff
- Round summary strip below header (non-scrolling, bg2): shows inspection type + date + agency + result badge — height ~56px
- Findings list: vertically scrollable, gap 8px, padding 16px/12px
- Default sort: 미조치(open) first, then 완료(resolved), within each group newest first
- Status filter tabs (미조치 / 완료 / 전체) via URL searchParam `tab` — matches RemediationPage pattern
- Tapping a finding card navigates to `/legal/:id/finding/:fid`
- Loading, empty, error states follow same patterns as Page 1

### Page 3: /legal/:id/finding/:fid — LegalFindingDetailPage (Finding Detail)

- Self-header (48px): title "지적 상세", ← back navigates to `/legal/:id`
- Layout: scrollable single column, 16px horizontal padding
- Section 1 — 지적 내용: description text (16px/700), location badge, before-photo (full-width, max-height 240px, object-fit cover, rounded 8px), created-by + date
- Section 2 — 조치 기록: only shown if status === 'resolved'; resolution memo + after-photo + resolved-by + resolved-at
- Section 3 — 조치 입력: shown only when status === 'open'; memo textarea + after-photo upload + "조치완료로 변경" primary button
- Admin-only actions (edit finding description/location): shown as small text link "수정" in header area, visible only when `role === 'admin'`
- PDF report link: shown in Page 1 card and in Page 2 round summary strip; opens new tab

### BottomSheet: Inspection Round Form (Admin only)

- Triggered by "+ 점검 등록" on LegalPage
- Overlay: `rgba(0,0,0,0.6)` backdrop, fixed fullscreen
- Sheet: `var(--bg2)`, radius 16px top, drag handle (32px×4px, `var(--bd2)`)
- Fields (top to bottom):
  1. 점검 유형 — horizontal 2-option toggle: 종합정밀점검 / 작동기능점검 (styled as tab selector, not native select)
  2. 점검일자 — `<input type="date">`, bg3, height 44px, full-width
  3. 점검기관 — `<input type="text">`, bg3, height 44px, full-width, placeholder "점검기관명 입력"
  4. 결과 — 3-option toggle: 적합 / 조건부적합 / 부적합
  5. 비고 — `<textarea>`, bg3, min-height 72px, full-width, optional
  6. 결과 보고서 — PDF upload button: "PDF 업로드" — file input accept=".pdf,application/pdf"; after upload shows filename + delete icon
- CTA: "점검 등록" primary button (full-width, height 48px, bg `var(--acl)`, text white 16px/700, rounded 10px)

### BottomSheet: Finding Form (All staff)

- Triggered by "+ 지적 등록" on LegalFindingsPage
- Same overlay + sheet style as above
- Fields:
  1. 지적 내용 — `<textarea>`, bg3, min-height 88px, full-width, required
  2. 위치 정보 — `<input type="text">`, bg3, height 44px, full-width, optional, placeholder "예: B1F 전기실 동쪽"
  3. 지적 사진 — photo upload button (uses usePhotoUpload hook); shows preview if captured
- CTA: "지적 등록" primary button (full-width, height 48px, bg `var(--acl)`, text white 16px/700, rounded 10px)

## State Definitions

Each view must handle these states explicitly:

| State | Trigger | Treatment |
|-------|---------|-----------|
| Loading | Query fetching | Skeleton cards (blink animation, bg3, radius 12) |
| Empty (no data) | Query success, result count === 0 | Centered: heading 16px/700/t1 + body 12px/400/t2 |
| Error | Query error or isError === true | Centered paragraph 14px/t2, retry instruction |
| Success | Query success, result count > 0 | Card list |
| Submitting (mutation) | Mutation in-flight | CTA button: disabled + "처리 중..." label |
| Submit success | Mutation success | toast.success() + invalidateQueries + close modal |
| Submit error | Mutation error | toast.error(message from API or fallback) |
| Admin-only UI | role !== 'admin' | "+ 점검 등록" button hidden; "수정" links hidden; PDF upload hidden |
