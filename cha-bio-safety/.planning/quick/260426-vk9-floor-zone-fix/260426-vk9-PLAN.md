# 260426-vk9: 지하층 floor 형식 + 소화기 zone 매핑 fix

## Context

직전 quick `260426-u7f` 로 CheckpointsPage 에서 카테고리=소화기 신규 등록 가능해진 후, 사용자가 첫 등록 시 두 가지 어긋난 데이터 발견:

1. floor 가 'B5F' 로 저장됨 → 기존 데이터(`B1`~`B5`)와 형식 불일치 → /floorplan "기존 개소 연결" 드롭다운에 안 보임
2. mgmt_no 가 '공-B5-01' 로 발급됨 → 기존 지하층 컨벤션('지-B5-NN')과 어긋남

## Root Cause

- `CheckpointsPage.tsx:97` `ZONE_FLOORS.common`: `'B1F','M','B2F','B3F','B4F','B5F'` 으로 잘못 정의 (FloorPlanPage/Floor 타입은 'B1'~'B5')
- `functions/api/extinguishers/create.ts`: 지하층(B prefix) 일 때 zone 을 '지' 로 매핑하는 처리 부재. UI 가 'common' 만 보내니 server 가 '공' 으로 매핑

## Tasks

### Task 1 — CheckpointsPage ZONE_FLOORS.common floor 형식 통일

`src/pages/CheckpointsPage.tsx:97`:
```diff
- common: ['B1F', 'M', 'B2F', 'B3F', 'B4F', 'B5F'],
+ common: ['B1', 'M', 'B2', 'B3', 'B4', 'B5'],
```

### Task 2 — extinguishers/create.ts 지하층 zone 강제

`functions/api/extinguishers/create.ts`:
- extFloor 변환 직후 `extZone` 파생: `/^B\d/i.test(extFloor) ? '지' : body.zone`
- mgmtPrefix 와 extinguishers INSERT 의 zone 컬럼 모두 `extZone` 사용
- check_points.zone 은 영문 enum 유지 (스키마 CHECK 제약 호환)
