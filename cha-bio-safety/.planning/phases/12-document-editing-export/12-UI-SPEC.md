## Color

Inherited from Phase 11 UI-SPEC. No new hex values. Reference vars only.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)` = #0d1117 | Main content area, page background, ExcelPreview outer container background |
| Secondary (30%) | `var(--bg2)` = #161b22 | Tab bar background, left panel background, list item default background |
| Tertiary surface | `var(--bg3)` = #1c2128 | List item hover, selected list item background |
| Quaternary surface | `var(--bg4)` = #22272e | Active/pressed list item background |
| Accent (10%) | `var(--acl)` = #3b82f6 | Active tab bottom border (2px) + active tab label, selected list item left border (3px) |
| Destructive | `var(--danger)` = #ef4444 | Not used in this phase |

Accent reserved for:
1. Active 대카테고리 tab — 2px bottom border + label color
2. Selected 항목 목록 item — 3px left border indicator (mirroring sidebar active pattern)

Accent is NOT used for: hover states, download/print buttons, focus rings, filter controls.

Border tokens:
- `var(--bd)` = rgba(255,255,255,0.07) — panel dividers (left panel right border, tab bar bottom border)
- `var(--bd2)` = rgba(255,255,255,0.13) — stronger section dividers within the left panel

ExcelPreview inner container (A4 paper):
- Background: `#ffffff` (white — not a CSS var, intentional for print context)
- Box shadow: `0 4px 24px rgba(0,0,0,0.3)` — visible only on screen, removed in `@media print`
- Border: none

## Component Inventory

### New Components (this phase)

#### `ExcelPreview` (`src/components/ExcelPreview.tsx`)

Renders a scaled-down read-only HTML table representation of a 점검일지 inside an A4-proportioned container.

- Outer container (`containerRef` div): `width: 100%`, `height: 100%`, `overflow: hidden`, `display: flex`, `alignItems: center`, `justifyContent: center`, `background: var(--bg)`
- Inner paper div (class `excel-preview-inner`): `width: 1123px`, `height: 794px` (A4 landscape), `background: #ffffff`, `boxShadow: 0 4px 24px rgba(0,0,0,0.3)`, `transformOrigin: center center`, `transform: scale({computed})`
- For portrait documents: `width: 794px`, `height: 1123px`
- Scale computed by ResizeObserver: `Math.min(containerWidth / A4_W, containerHeight / A4_H) * 0.95`
- Table cells: 11px, black text, `border: 1px solid #ccc`, `padding: 2px 4px`
- Merged cells: `colSpan` / `rowSpan` HTML attributes
- Header cells: background `#e8e8e8`, weight 700
- No scrollbar on the outer container — scale ensures full fit

Props interface:
```
type: ReportType
year: number
month?: number   // optional — some reports are annual
```

Loading state: centered spinner using `var(--acl)` color, 24px diameter
Empty/no-data state: centered text "선택한 기간의 점검 데이터가 없습니다." in 14px `var(--t2)`

#### `DesktopReportsPage` (inline in `ReportsPage.tsx` or `src/pages/ReportsPage.tsx` isDesktop branch)

The 3-panel desktop layout for 점검일지 조회/출력 (D-01, D-11).

Structure:

```
├─────────────────────────────┬────────────────────────────────────────┤
│  항목 목록 (280px fixed)    │  A4 미리보기 영역 (flex: 1)            │
│  [data-no-print]            │                                        │
│  ─ 필터: 연도 / 월 ─        │   ┌──────────────────────────────┐    │
│  • 월초 유수검지 장치 점검표 │   │  ExcelPreview (scaled A4)   │    │
│  • 월말 유수검지 장치 점검표 │   │  white paper, table         │    │

└─────────────────────────────┴────────────────────────────────────────┘
```

- Outer container: `display: flex; flexDirection: column; height: 100%`
- Tab row: `display: flex; borderBottom: 1px solid var(--bd); background: var(--bg2); flexShrink: 0; height: 44px; alignItems: stretch`
- Body row: `flex: 1; display: flex; overflow: hidden`
- Left panel: `width: 280px; flexShrink: 0; borderRight: 1px solid var(--bd); overflow: auto; display: flex; flexDirection: column; background: var(--bg2)`
- Right panel: `flex: 1; overflow: hidden; background: var(--bg)`

**Tab row items:**
- Inactive: `padding: 0 20px; fontSize: 14px; color: var(--t2); background: transparent; border: none; cursor: pointer`
- Active: same + `borderBottom: 2px solid var(--acl); color: var(--acl)`
- Hover (inactive only): `color: var(--t1)` (instant, no transition)

**Left panel sections:**

Top: 연도/월 필터 row

- `<select>` for month (optional — shown only for monthly reports): same style

Middle: 항목 목록 (scrollable, `flex: 1; overflow: auto`)

- Hover: `background: var(--bg3)` (instant)
- Selected: `background: var(--bg3); borderLeft: 3px solid var(--acl)`
- Non-selected left border: `3px solid transparent` (prevents layout shift on selection — mirrors DesktopSidebar pattern)
- Item title: 14px, weight 400, `var(--t1)`, single line, `overflow: hidden; textOverflow: ellipsis; whiteSpace: nowrap`
- Item subtitle: 12px, weight 400, `var(--t2)`, below title

Bottom: action button row
- `padding: 12px 16px; display: flex; gap: 8px; flexShrink: 0; borderTop: 1px solid var(--bd)`
- 엑셀 다운로드 button: `flex: 1; height: 36px; background: var(--bg3); border: 1px solid var(--bd2); borderRadius: 6px; color: var(--t1); fontSize: 14px; cursor: pointer`
- 인쇄 button: `flex: 1; height: 36px; background: var(--bg3); border: 1px solid var(--bd2); borderRadius: 6px; color: var(--t1); fontSize: 14px; cursor: pointer`
- Both buttons hover: `background: var(--bg4)` (instant)
- Both buttons icons: Download (16px) and Printer (16px) from lucide-react, 4px gap before label

```css
@media print {
  /* Hide all non-print UI */
  [data-no-print] {
    display: none !important;
  }

  /* Print-only elements shown */
  [data-print-only] {
    display: block !important;
  }

  /* A4 landscape for 점검일지 */
  @page {
    size: A4 landscape;
    margin: 0;
  }

  /* Remove scale transform, show full A4 */
  .excel-preview-inner {
    transform: none !important;
    width: 210mm !important;
    box-shadow: none !important;
    position: fixed !important;
    inset: 0 !important;
  }

  body {
    background: white !important;
  }
}
```

`data-no-print` is applied to: DesktopSidebar root div, desktop header div, left panel (항목 목록), tab row, action button row.

`excel-preview-inner` class is applied to: the inner A4 div inside ExcelPreview.

### Wide Table/Card Layout (LAYOUT-02)

For pages using wide data display on desktop (e.g., 항목 목록 inside the left panel):
- Row height: 44px minimum
- Full available width: `width: 100%`
- Header row: `background: var(--bg3)`, 12px weight 700 `var(--t2)` uppercase
- Body row: `background: var(--bg2)`, hover `background: var(--bg3)`
- Row border: `borderBottom: 1px solid var(--bd)`

This is the same Table Layout pattern from Phase 11 UI-SPEC — applied to document list rows.

### Print Layout (DOC-04)

On `window.print()` / Ctrl+P:
1. `[data-no-print]` elements: `display: none !important`
2. `.excel-preview-inner`: `transform: none !important; width: 210mm !important; position: fixed !important; inset: 0 !important; box-shadow: none !important`
3. `@page { size: A4 landscape; margin: 0 }`
4. `body { background: white !important }`

Result: only the A4 HTML table fills the printed page with no chrome around it.
