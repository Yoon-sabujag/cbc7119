# Phase 11 — UI Design Contract

> Visual and interaction contract for Phase 11: Desktop Layout Foundation.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Source: CONTEXT.md (18 decisions), RESEARCH.md (stack + patterns), index.css (existing tokens).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — CSS Custom Properties + Tailwind CSS 3.4 |
| Preset | not applicable |
| Component library | none (hand-rolled, inline styles + CSS vars) |
| Icon library | lucide-react 0.454.0 |
| Font | Noto Sans KR (primary), JetBrains Mono (code/numbers) |

No shadcn. This project uses CSS Custom Properties (`var(--bg)`, `var(--t1)`, etc.) applied via inline React styles and Tailwind token aliases. All new components for this phase must follow the same pattern — inline styles using CSS vars, no new libraries.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline icon-to-label gap inside sidebar nav items |
| sm | 8px | Compact row padding, section header vertical padding |
| md | 16px | Default element padding — sidebar item horizontal padding, header horizontal padding |
| lg | 24px | Sidebar section separator gap, card internal padding |
| xl | 32px | Main content area horizontal padding on desktop |
| 2xl | 48px | Large section breaks (e.g. space between KPI card groups) |
| 3xl | 64px | Page-level top/bottom spacing when no header is present |

Exceptions:
- Sidebar width: 280px fixed (from D-03 — not a spacing token, a layout dimension)
- Desktop header height: 48px fixed
- Mobile BottomNav height: `calc(54px + var(--sab, 34px))` — existing, do not change
- Sidebar nav item height: 36px (minimum touch-safe for mouse cursor; desktop-only)
- Sidebar user card at bottom: 56px height with 16px horizontal padding

## Typography

Source: existing `index.css` + Tailwind config. All sizes apply to desktop layout components only; mobile pages are untouched.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 13px | 500 (medium) | 1.4 |
| Heading (page title in desktop header) | 13px | 700 (bold) | 1.0 |
| Section header (sidebar group label) | 11px | 600 (semibold) | 1.2 |

Notes:
- Sidebar nav item labels: 14px, weight 500, truncated to single line with `text-overflow: ellipsis`
- KPI card values on dashboard: 28px, weight 700 (existing dashboard pattern — do not change)
- KPI card labels: 12px, weight 400 — existing pattern
- Korean text only: Noto Sans KR. No mixed monospace in navigation or headers.

## Color

All values are the existing CSS Custom Properties from `src/index.css`. Do not introduce new hex values — reference vars only.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)` = #0d1117 | Main content area background, page backgrounds |
| Secondary (30%) | `var(--bg2)` = #161b22 | Sidebar background, desktop header background |
| Tertiary surface | `var(--bg3)` = #1c2128 | Sidebar nav item hover state, card backgrounds |
| Quaternary surface | `var(--bg4)` = #22272e | Active sidebar nav item background |
| Accent (10%) | `var(--acl)` = #3b82f6 | Active sidebar nav item left border indicator (3px), active nav item label color |
| Destructive | `var(--danger)` = #ef4444 | Logout button text color only |

Accent reserved for:
1. Active sidebar nav item — 3px left border in `var(--acl)`
2. Active sidebar nav item label text color

Accent is NOT used for: hover states, focus rings, badges, header elements, or any other interactive element in this phase.

Border tokens in use:
- `var(--bd)` = rgba(255,255,255,0.07) — sidebar/header divider lines
- `var(--bd2)` = rgba(255,255,255,0.13) — section separator lines within sidebar

Text tokens:
- `var(--t1)` = #e6edf3 — primary text (nav labels, page title, user name)
- `var(--t2)` = #8b949e — secondary text (section group labels, user role label)
- `var(--t3)` = #6e7681 — tertiary text (nav item badge "soon" labels)

## Component Inventory

### New Components (this phase)

#### `DesktopSidebar` (`src/components/DesktopSidebar.tsx`)

- Width: 280px, fixed, `flex-shrink: 0`
- Height: `100dvh` (full viewport height)
- Background: `var(--bg2)`
- Right border: `1px solid var(--bd)`
- Layout: flex column — top logo strip (48px) + scrollable nav area (flex: 1) + user card footer (56px)

**Logo strip (top 48px):**
- Background: `var(--bg2)`
- Bottom border: `1px solid var(--bd)`
- Content: App name "CBC 방재" in 14px, weight 700, `var(--t1)` — left-aligned with 16px left padding
- No icon, no hamburger

**Nav sections (scrollable middle):**
- Overflow: `auto` (scrollable if content overflows)
- Section group label: 11px, weight 600, `var(--t2)`, uppercase, 16px left padding, 8px top + 4px bottom vertical padding
- Section items: 36px height, 16px horizontal padding, 14px weight 500 `var(--t1)` label
- Item hover: background `var(--bg3)`, no transition delay (instant)
- Item active: background `var(--bg4)`, 3px left border `var(--acl)`, label color `var(--acl)`
- Item with `soon: true`: label color `var(--t3)`, cursor `default`, pointer-events none
- Unresolved badge on "조치 관리": 16px diameter pill, `var(--danger)` background, white 11px weight 700 count

**Desktop section order (re-mapped from D-04):**

Section 1: 점검 현황
- 대시보드 (`/dashboard`)
- 소방 점검 (`/inspection`)
- 조치 관리 (`/remediation`) — badge for unresolved count
- 건물 도면 (`/floorplan`)

Section 2: 문서 관리

Section 3: 직원 관리
- 근무표 (`/workshift`)
- 연차 및 식사 (`/staff-service`)

Section 4: 시설 관리
- 승강기 관리 (`/elevator`)

- DIV 압력 관리 (`/div`)
- QR 스캔 (`/inspection/qr`)
- 관리자 설정 (`/admin`) — role: 'admin' only (hide for 'assistant')

**User card footer (56px):**
- Background: `var(--bg2)`
- Top border: `1px solid var(--bd)`
- Layout: 16px horizontal padding, flex row, align center
- Left: user name 13px weight 600 `var(--t1)` + role label 11px `var(--t2)` stacked vertically
- Right: logout icon button (LogOut from lucide-react, 16px, `var(--danger)`)
- Logout on click: call `logout()` from authStore, no confirmation dialog (not destructive data-wise)

#### `useIsDesktop` hook (`src/hooks/useIsDesktop.ts`)

- Uses `window.matchMedia('(min-width: 1024px)')`
- SSR-safe: initializes with `typeof window !== 'undefined'` guard
- Returns boolean, updates on window resize

### Modified Components

#### `App.tsx` — Layout function

- Add `isDesktop = useIsDesktop()` at top
- When `isDesktop && showNav`: render `<DesktopSidebar>` + slim desktop header (48px)
- When `!isDesktop && showNav`: render existing `<GlobalHeader>` + `<SideMenu>` + `<BottomNav>`
- `<main>` paddingBottom: `(!isDesktop && showNav) ? 'calc(54px + var(--sab, 34px))' : 0`
- Outer div: `overflow: hidden` removed from html global (see below)

#### `src/index.css` — line 42 `html { overflow: hidden }`

Change to:
```css
@media (max-width: 1023px) {
  html {
    overflow: hidden;
  }
}
```
This is the critical blocker fix from STATE.md. Desktop viewport must scroll; mobile must stay locked.

#### `SettingsPanel.tsx` — bottom position

- Current: `bottom: 'calc(54px + var(--sab, 34px) - var(--sat, 0px))'`
- Desktop override: when `isDesktop` prop is true, set `bottom: 0`
- Add `isDesktop?: boolean` prop to SettingsPanel interface

#### `GlobalHeader.tsx` — no change needed

- Hidden on desktop via conditional render in Layout, not via CSS. No modification required.

#### `SideMenu.tsx` — export MENU constant

- Add `export` keyword to the `MENU` constant so `DesktopSidebar` can import it
- No other changes

## Layout Patterns

### Desktop Shell (>= 1024px)

```
┌─────────────────────────────────────────────────────┐
│ DesktopSidebar (280px)  │ Desktop Header (48px tall) │
│ [Logo strip 48px]       │ [Page title    ] [Settings]│
│ ─────────────────────── ├──────────────────────────  │
│ Section: 점검 현황      │                            │
│   대시보드              │   <main> (overflow: auto)  │
│   소방 점검             │   Page content here        │
│   조치 관리  [3]        │   Full width, scrollable   │
│   건물 도면             │                            │
│ Section: 문서 관리      │                            │
│   ...                   │                            │
│ ─────────────────────── │                            │
│ [User name  ] [Logout]  │                            │
└─────────────────────────────────────────────────────┘
```

### Multi-Panel Layout (문서목록+미리보기, 조치목록+상세)

Applied at page level, not shell level. Pages that implement D-14 use:
- `display: flex; flex-direction: row; height: 100%`
- Left panel: 360px fixed width, `overflow: auto`, `border-right: 1px solid var(--bd)`
- Right panel: `flex: 1`, `overflow: auto`

This phase scaffolds the shell. Specific multi-panel pages are Phase 12 responsibility.

### Single Panel (넓게) Layout

For 대시보드, 근무표, 승강기, 관리자 설정:
- `max-width: none` — use full available width
- `padding: 24px 32px` — lg vertical + xl horizontal

### Tab Switcher (목록 뷰 / 도면 뷰) — D-16

For inspection results, floor plan tabs:
- Tab bar: `display: flex`, `border-bottom: 1px solid var(--bd)`, `background: var(--bg2)`
- Tab item: 14px weight 500, 16px horizontal + 10px vertical padding
- Active tab: `border-bottom: 2px solid var(--acl)`, label color `var(--acl)`
- Inactive tab: label color `var(--t2)`, no border
- Floor filter tabs (B2, B1, 1F, 2F, 3F, 전체): same visual style, rendered as pill row below main tabs

### Table Layout (D-18)

- Width: `width: 100%`
- Header row: `background: var(--bg3)`, 12px weight 600 `var(--t2)` uppercase label
- Body row: `background: var(--bg)`, hover `background: var(--bg3)`
- Row height: 44px minimum
- Row border: `border-bottom: 1px solid var(--bd)`
- Row click for accordion expand: chevron-right icon rotates 90deg on expand (no animation duration — instant)

### Accordion Expand (D-11)

- On row click: expand child rows inline below the parent
- Expanded area: `background: var(--bg3)`, 16px left indent
- Collapse/expand: no CSS transition for this phase (instant toggle) — avoids complexity risk

### KPI Cards (D-17, dashboard)

- 4 cards in a single `display: grid; grid-template-columns: repeat(4, 1fr); gap: 16px`
- Card: `background: var(--bg2)`, `border: 1px solid var(--bd)`, `border-radius: 8px`, `padding: 24px`
- Value: 28px weight 700 `var(--t1)`
- Label: 12px weight 400 `var(--t2)`

## Copywriting Contract

This phase has no new data-entry forms or destructive flows. Copywriting scope is limited to structural UI labels.

| Element | Copy |
|---------|------|
| App name in sidebar logo strip | CBC 방재 |
| Sidebar section 1 label | 점검 현황 |
| Sidebar section 2 label | 문서 관리 |
| Sidebar section 3 label | 직원 관리 |
| Sidebar section 4 label | 시설 관리 |
| Logout button tooltip / aria-label | 로그아웃 |
| User role — admin | 관리자 |
| User role — assistant | 부관리자 |
| Empty state (no data in table) | 데이터가 없습니다 |
| Error state (layout load failure) | 페이지를 불러오지 못했습니다. 새로고침 해주세요. |
| Destructive action (logout) | 별도 확인 없음 — 즉시 실행 (세션 손실 없음, 인증 토큰만 삭제) |

No primary CTA for this phase — Phase 11 is purely structural (layout shell). Page-level CTAs are unchanged from existing implementations.
